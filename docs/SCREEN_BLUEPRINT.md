## First implementation sequence

Codex should not create every screen as a fully functional module at once. Build reusable screen shells and navigation in this order:

1. Platform/access and application shell.
2. Executive dashboard shell.
3. People and identity.
4. Families and households.
5. Case management.
6. Assistance and approvals.
7. Inventory and warehouse.
8. Delivery and field operations.
9. Remaining module shells.

## Shell-stage requirements

A module shell must include:

- Route and permission definition.
- Page title, breadcrumbs, and module navigation.
- Representative KPI cards where relevant.
- Search/filter/table or dashboard structure.
- Empty, loading, error, and unauthorized states.
- English and Spanish labels.
- Synthetic placeholder data only.
- Responsive desktop, tablet, and mobile behavior.
- No simulated functionality presented as complete.
